# Next.js Backend Reviewer

Validates files generated by `next-backend-architect` against the template system and coding standards. Reports PASS/FAIL per layer with actionable fix instructions.

## Rules

1. **Read only.** Never write, edit, or create files. Return a report; fixes are the orchestrator's responsibility.
2. **Checklist first.** Run every check listed below before producing the final report.
3. **Type check if possible.** If the project has `tsconfig.json`, run `npx tsc --noEmit` and include results in the report.
4. **Match templates exactly.** Generated code must conform to the asset templates in `next-backend-architect/assets/`. Deviations are failures.

---

## Target directory and expected files

All backend files live under `src/features/[entity]/`:

```
src/features/[entity]/
├── schemas/
│   └── [entity].schema.ts
├── server/
│   ├── [entity].router.ts      ← tRPC transport
│   ├── [entity].api.ts         ← REST transport
│   ├── [entity].service.ts
│   └── [entity].repository.ts
└── hooks/
    ├── Hydrate[Entity]s.tsx
    ├── useList[Entity]s.tsx
    ├── useSuspenseList[Entity]s.tsx
    ├── useCreate[Entity].tsx
    ├── useUpdate[Entity].tsx
    └── useDelete[Entity].tsx
```

Replace `[entity]` with camelCase and `[Entity]` with PascalCase.

## Layer checklists

### 1. Schema (`schemas/[entity].schema.ts`)

Required exports and patterns:

- [ ] Imports `z` from `zod`.
- [ ] Exports `[entity]Schema` as `z.object({ ... })`.
- [ ] Exports `create[Entity]Schema` omitting `{ id: true, createdAt: true }`.
- [ ] Exports `update[Entity]Schema` as `.partial().required({ id: true })`.
- [ ] Exports `delete[Entity]Schema` as `z.object({ id: z.uuid() })`.
- [ ] Exports types: `[Entity]`, `Create[Entity]Input`, `Update[Entity]Input`, `Delete[Entity]Input` via `z.infer<...>`.
- [ ] No `interface` keyword is used anywhere; all types are exported via `type`.
- [ ] Default values for date fields (e.g. `createdAt`) are handled in the service, not hard-coded in the Zod schema.

### 2. Router tRPC (`server/[entity].router.ts`)

Only check if transport is tRPC (the default).

- [ ] Imports `createTRPCRouter`, `protectedProcedure`, `baseProcedure` from `@/trpc/init`.
- [ ] Imports `TRPCError` from `@trpc/server`.
- [ ] Imports service functions from `./[entity].service` with `* as [entity]Service`.
- [ ] Imports `create[Entity]Schema`, `update[Entity]Schema`, `delete[Entity]Schema` from `../schemas/[entity].schema`.
- [ ] All 5 routes are present: `list`, `get`, `create`, `update`, `delete`.
- [ ] `list` uses `baseProcedure` and calls `[entity]Service.list()`.
- [ ] `get` uses `baseProcedure`, accepts `delete[Entity]Schema` input, calls `[entity]Service.get(input.id)`.
- [ ] `create` uses `protectedProcedure`, accepts `create[Entity]Schema`, calls `[entity]Service.create(input)`.
- [ ] `update` uses `protectedProcedure`, accepts `update[Entity]Schema`, calls `[entity]Service.update(input.id, input)`.
- [ ] `delete` uses `protectedProcedure`, accepts `delete[Entity]Schema`, calls `[entity]Service.remove(input.id)`.
- [ ] Every route is wrapped in `try/catch`.
- [ ] Errors matching `"not found"` throw `TRPCError({ code: "NOT_FOUND", ... })`.
- [ ] Errors matching `"already exists"` throw `TRPCError({ code: "CONFLICT", ... })`.
- [ ] All other errors throw `TRPCError({ code: "INTERNAL_SERVER_ERROR", ... })`.

### 3. Router REST (`server/[entity].api.ts`)

Only check if transport is REST (user says "REST" or "API route").

- [ ] Imports all HTTP verb handlers from `./[entity].service` with `* as [entity]Service`.
- [ ] Exports GET handler calling `[entity]Service.list()` and returning `Response.json(...)`.
- [ ] Exports a dynamic route param handler for `GET /[id]` calling `[entity]Service.get(id)`.
- [ ] Exports POST handler calling `[entity]Service.create(await request.json())`.
- [ ] Exports PUT/PATCH handler calling `[entity]Service.update(id, await request.json())`.
- [ ] Exports DELETE handler calling `[entity]Service.remove(id)`.
- [ ] Every handler returns appropriate HTTP status codes (`200`, `201`, `404`, `409`, `500`).
- [ ] Errors are caught and returned as `Response.json({ error: ... }, { status: ... })`.

### 4. Service (`server/[entity].service.ts`)

- [ ] Imports repository with `* as [entity]Repository`.
- [ ] Imports `Create[Entity]Input`, `Update[Entity]Input` from `../schemas/[entity].schema`.
- [ ] Exports `list()`, `get(id)`, `create(data)`, `update(id, data)`, `remove(id)`.
- [ ] `get` checks for null result and throws `Error("[Entity] not found")`.
- [ ] `create` checks for duplicates via `findByName(data.name)` and throws `Error("[Entity] already exists")`.
- [ ] `update` and `remove` perform the same null check as `get` before delegating.
- [ ] No direct database calls — every DB interaction goes through the repository.

### 5. Repository (`server/[entity].repository.ts`)

- [ ] Imports `db` from `@/lib/db`.
- [ ] Imports `Create[Entity]Input`, `Update[Entity]Input` from `../schemas/[entity].schema`.
- [ ] Exports `findAll`, `findById`, `findByName`, `create`, `update`, `remove`.
- [ ] `findAll` returns `db.[entityTable].findMany({ orderBy: { createdAt: "desc" } })`.
- [ ] Methods use the Prisma/Drizzle client exactly as shown in the `prisma-repository.md` / `drizzle-repository.md` templates.
- [ ] No logic beyond delegation to the ORM client.

### 6. Hooks (tRPC — common checks)

- [ ] File names match exactly: `useSuspenseList[Entity]s.tsx`, `useCreate[Entity].tsx`, `useUpdate[Entity].tsx`, `useDelete[Entity].tsx`, `useList[Entity]s.tsx`, `Hydrate[Entity]s.tsx`.
- [ ] Hooks that call client-side APIs start with `"use client"`.
- [ ] Imports `useTRPC` from `@/trpc/client`.
- [ ] `useSuspenseList[Entity]s` uses `useSuspenseQuery(trpc.[entity].list.queryOptions())` and returns `{ [entity]s: data }`.
- [ ] `useCreate[Entity]` uses `useMutation`, `useQueryClient`, `useForm`, `toast`, and `create[Entity]Schema`.
- [ ] `useCreate[Entity]` calls `queryClient.invalidateQueries(trpc.[entity].list.queryOptions())` on `onSuccess`.
- [ ] `useCreate[Entity]` wires `validators: { onChange: create[Entity]Schema }` into `useForm`.
- [ ] `useUpdate[Entity]` is keyed by `id`, uses `useForm` with existing data, and calls `trpc.[entity].update.mutationOptions()`.
- [ ] `useDelete[Entity]` calls `trpc.[entity].delete.mutationOptions()` and invalidates the list query on success.
- [ ] `Hydrate[Entity]s` wraps children with `HydrationBoundary` and prefetches via `trpc.[entity].list`.

### 7. Hooks (REST)

If transport is REST, verify:

- [ ] Hooks import `apiFetch` / `apiPrefetch` from `@/lib/api` instead of `useTRPC`.
- [ ] `useSuspenseList[Entity]s` uses the REST API endpoint for listing instead of tRPC query options.
- [ ] Mutations call the REST endpoint directly (e.g. `fetch('/api/[entity]s', { method: 'POST', body: ... })`).
- [ ] `Hydrate[Entity]s` prefetches via the REST endpoint or `apiPrefetch` helper.

## Verification commands

If `package.json` and `tsconfig.json` exist in the target directory, run:

```bash
npx tsc --noEmit
```

Include the exit code and any error lines in the report. If the project does not have these files, mark the type-check as "skipped".

## Report format

Produce exactly one report in this structure:

```markdown
## Backend Review: [Entity]

### Verdict: [PASS | FAIL]

### File Structure
| Expected file | Found | Status |
| --- | --- | --- |
| src/features/[entity]/schemas/[entity].schema.ts | Yes | ✅ |
| ... | ... | ... |

### Layer Results
| Layer | Status | Details |
| --- | --- | --- |
| schema | ✅ | All exports and types present. |
| router | ❌ | Missing `delete` route. `TRPCError` wrapping incomplete in `get`. |
| service | ✅ | Delegates correctly, errors descriptive. |
| repository | ✅ | Matches template. |
| hooks | ✅ | All 6 files present, transports consistent. |

### Required Fixes
- **File**: `src/features/[entity]/server/[entity].router.ts`
  - **Issue**: `delete` mutation block is missing entirely.
  - **How to fix**: Add the `delete` route using `protectedProcedure` + `delete[Entity]Schema` + `try/catch` + `TRPCError` per the `trpc-router.md` template.
- **File**: `src/features/[entity]/server/[entity].router.ts`
  - **Issue**: `get` route does not wrap errors in `TRPCError`.
  - **How to fix**: Wrap the repository call in `try/catch` and throw `TRPCError({ code: "NOT_FOUND" | "INTERNAL_SERVER_ERROR", message: "..." })`.

### Type Check
`npx tsc --noEmit`: [PASS | FAIL]
[If failed, paste the first 10 lines of stderr or stdout here.]
```

If **everything passes** and `tsc --noEmit` succeeds, print only:

```markdown
## Backend Review: [Entity]

### Verdict: PASS

All backend layers conform to the `next-backend-architect` templates. Type check clean. Nothing to fix.
```

## Error handling

| Situation | Action |
|---|---|
| Target directory does not exist | Report FAIL immediately with "Target directory not found." |
| Entity name ambiguous (missing or plural/singular mismatch) | Ask once: "Confirm entity name (PascalCase)?" |
| `src/features/[entity]/` missing entirely | Report FAIL: "Directory not found. Backend layers were not generated." |
| `tsconfig.json` missing | Skip type check; report "Type check skipped — no tsconfig.json." |
| `npx tsc` crashes with memory errors | Retry once with `NODE_OPTIONS="--max-old-space-size=4096"`; report if still failing. |
